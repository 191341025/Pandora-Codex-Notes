## 概念讲解

### 1. Prompt Engineering（提示工程）

你给 LLM 的每一段输入文字就是一个 Prompt（提示）。Prompt Engineering 就是**设计高质量提示来引导 LLM 产出准确输出的过程**。

为什么这件事值得专门研究？因为 LLM 本质上是一个**预测引擎**——它拿到你的输入文本后，逐个预测"下一个 token 最可能是什么"，然后把预测出的 token 接到末尾，继续预测下一个，循环往复。你写的 prompt 越清晰，LLM 就越容易沿着正确方向预测。

这里的关键洞察是：**LLM 不是"理解"你的问题然后"思考"答案，而是在预测最可能跟在你输入后面的文本序列**。所以 prompt 的措辞、结构、上下文、语气——每一个细节都会影响预测方向。一个模糊的 prompt 就像给 GPS 输入了一个模糊的目的地，它可能把你带到完全不同的地方。

Prompt Engineering 是一个**迭代过程**：写 prompt → 看输出 → 调整 prompt → 再试。影响 prompt 效果的因素包括：

| 因素 | 说明 |
|------|------|
| 模型选择 | 不同模型（Gemini、GPT、Claude、Gemma、LLaMA）对同一 prompt 反应不同 |
| 模型配置 | Temperature、Top-K、Top-P 等参数直接影响输出风格 |
| 用词和语气 | "Explain like I'm 5" vs "Provide a technical analysis" 得到完全不同的结果 |
| 结构和格式 | 有没有示例、有没有明确的输出格式要求 |
| 上下文信息 | 给模型多少背景知识 |

> **什么时候用**：只要你在和 LLM 交互——不管是文本摘要、信息提取、问答、分类、翻译、代码生成还是推理——你都在做 prompt engineering。区别只在于你是随手写还是系统化地优化。

### 2. Temperature（温度）

Temperature 是你最常用的模型参数，它控制**输出的随机程度**。

**直觉建立**：

想象你面前有一个装满彩球的箱子，每个球代表一个可能的下一个 token，球的大小代表它的概率——概率越高，球越大。

- **Temperature = 0（贪心解码）**：你闭着眼睛伸手进去，但手只会抓到最大的那个球。每次都是。结果完全确定，没有任何随机性。（边界情况：如果恰好有两个球一样大——即两个 token 概率完全相同——取决于实现的 tie-breaking 机制，输出可能不完全一致。）
- **Temperature 很低（比如 0.1-0.2）**：大球还是占优势，但偶尔你也会摸到第二大、第三大的球。输出基本稳定，偶有小变化。
- **Temperature 接近 1**：球的大小差异被"压平"了，中等大小的球也有不小的机会被摸到。输出变得多样、有创意。
- **Temperature 很高（>1）**：所有球几乎变成一样大。随便哪个 token 都可能被选中，输出变得不可预测。

> **类比边界**：这个类比没有体现 temperature 对 softmax 函数的数学作用——实际上 temperature 是 softmax 的除数，低 temperature 让概率分布更"尖锐"（赢者通吃），高 temperature 让分布更"平坦"（雨露均沾）。

**具体推荐值**：

| 场景 | Temperature | 理由 |
|------|-------------|------|
| 数学题、事实问答 | 0 | 只有一个正确答案，不需要随机性 |
| 一般任务 | 0.1-0.2 | 基本确定但允许微小变化 |
| 创意写作、头脑风暴 | 0.7-0.9 | 需要多样性和意外惊喜 |
| 极端创意探索 | >1 | 输出可能很随机，慎用 |

> **常见误用**：把 temperature 调到很高然后抱怨"模型在胡说八道"。高 temperature 不是让模型"更聪明"或"更有创意地思考"，只是让它在选词时更随机。如果你的任务需要准确性，高 temperature 只会帮倒忙。

### 3. Top-K 与 Top-P 采样

LLM 在预测下一个 token 时，并不是只给出一个答案——它为词汇表中的**每个 token** 都计算了一个概率。Top-K 和 Top-P 是两种从这些概率中**筛选候选 token** 的方法。

**Top-K 采样**：只保留概率最高的 K 个 token，从中选一个。

| Top-K 值 | 效果 |
|----------|------|
| 1 | 等于贪心解码——只选最高概率的那个 |
| 20 | 比较保守，输出偏稳定、偏事实 |
| 40 | 中等创意，允许更多词汇选择 |
| 很大（接近词汇表大小） | 基本没有过滤效果 |

**Top-P 采样**（也叫 Nucleus Sampling）：从概率最高的 token 开始累加，直到累积概率达到 P 值，保留这些 token。

| Top-P 值 | 效果 |
|----------|------|
| 接近 0 | 只看最高概率的一两个 token |
| 0.9 | 保留累积概率达 90% 的候选集 |
| 0.95 | 稍宽松，更多样 |
| 1 | 所有非零概率 token 都是候选 |

**Top-K vs Top-P 的选择**：没有绝对的优劣之分。Top-K 固定候选数量，Top-P 动态调整（概率分布尖锐时候选少，平坦时候选多）。实际使用中往往两个一起设，让模型同时满足两个条件。

### 4. Output Length（输出长度）

设置模型最多生成多少个 token。这个参数看似简单，但有两个容易踩的坑：

**坑一：截断 ≠ 精简**。设了 max_tokens=100 不代表模型会"用 100 个 token 精炼地回答"，而是模型写到第 100 个 token 时被强制停止——哪怕话说到一半。如果你想要简短的输出，需要在 prompt 里明确说，比如"用一句话回答"或"控制在 50 字以内"。

**坑二：成本和延迟**。生成的 token 越多，消耗的算力越大，响应越慢，费用越高。不要无脑设大值。

> 这个参数对某些技术特别重要：比如 ReAct 模式下，LLM 在给出有用回答后可能继续输出无意义的填充文本，此时用 output length 限制可以有效截断。

### 5. 参数组合策略：三个旋钮怎么配合

Temperature、Top-K、Top-P 这三个参数**互相影响**，理解它们的交互关系才能有效调参。

**执行顺序**（以 Vertex AI 为例）：
1. 先用 Top-K 和 Top-P **筛选候选 token**（取交集，两个条件都要满足）
2. 再对通过筛选的 token 应用 **Temperature** 进行采样

**极端值的覆盖效应**：

| 设置 | 效果 |
|------|------|
| Temperature = 0 | Top-K 和 Top-P 失效——直接选最高概率的 token |
| Top-K = 1 | Temperature 和 Top-P 失效——只有一个候选，没得选 |
| Top-P ≈ 0 | Temperature 和 Top-K 失效——同上 |
| Temperature 极高 | Temperature 失效——通过筛选的 token 被随机等概率选取 |
| Top-K 极大 / Top-P = 1 | 该参数失效——不起过滤作用 |

**推荐起步配置**：

| 目标 | Temperature | Top-P | Top-K |
|------|-------------|-------|-------|
| 一般任务（平衡） | 0.2 | 0.95 | 30 |
| 创意任务 | 0.9 | 0.99 | 40 |
| 保守/事实任务 | 0.1 | 0.9 | 20 |
| 确定性答案（数学等） | 0 | — | — |

> **常见误用**：三个参数各调各的，不理解它们的交互。比如 Temperature 已经设了 0，还去纠结 Top-P 该设多少——没意义，因为 Temperature=0 已经覆盖了其他两个参数。

### 6. 重复循环 Bug（Repetition Loop）

一个你迟早会遇到的问题：模型输出大量重复的填充词，同一个词或句式反复出现，直到输出窗口被填满。

**为什么会发生？**

这和 temperature 及 Top-K/Top-P 的设置有关，但有趣的是，**过高和过低都可能触发**：

- **Temperature 过低**：模型变得过于确定性，如果概率最高的路径恰好指向之前已生成的文本，就会陷入循环
- **Temperature 过高**：输出过于随机，但随机选中的词偶然触发了之前的模式，同样可能陷入循环

**怎么解决**：没有一劳永逸的方案，需要在 temperature 和 Top-K/Top-P 之间反复微调，找到"确定性"和"随机性"之间的平衡点。这也是为什么前面推荐从中等值（Temperature 0.2, Top-P 0.95, Top-K 30）开始，而不是极端值。

---

## 重点标记

1. **LLM 是预测引擎**：它预测"下一个最可能的 token"，不是"理解问题然后思考答案"。Prompt 的每个细节都影响预测方向。
2. **Temperature 控制随机性**：0 = 确定性输出，越高越随机。不要指望高 temperature 让模型"更聪明"。
3. **Top-K 和 Top-P 是候选过滤器**：限制模型从多少个 token 中选择，和 Temperature 配合使用。
4. **参数有覆盖效应**：某个参数设到极端值时，其他参数可能失效。理解交互关系比单独调参重要。
5. **Output Length 是硬截断**：它不会让模型"说得更精炼"，只会在到达限制时强制停止。

## 自测：你真的理解了吗？

**Q1**：你需要让 LLM 对用户评论做情感分类（正面/负面/中性），要求每次对同一条评论给出一致的结果。你会怎么设 Temperature？为什么？

**Q2**：你把 Temperature 设为 0，Top-K 设为 40，Top-P 设为 0.95。实际效果是什么？Top-K 和 Top-P 还起作用吗？

**Q3**：你设了 max_tokens=50，然后 prompt 写的是"写一篇关于量子物理的博客文章"。输出很可能会出什么问题？你应该怎么改？

**Q4**：你在做一个创意写作任务，Temperature 设为 1.5，结果模型输出了大量重复的句子。你猜可能是什么原因？你会怎么调？

**Q5**：同事说"Top-K 设大一点输出就更有创意"。这句话对吗？在什么条件下可能不成立？
