# 模块五：企业管理、限制与故障排除

> 对应 PDF 第 13-15 页

---

## 概念讲解

### 1. Team/Enterprise 管理员控制

**管理开关**：组织管理员可以通过 Admin 设置中的一个简单开关来启用或禁用 Cowork。

**当前限制**：
- 这是一个**组织级别的全局开关**——要么全开，要么全关
- **没有按用户或角色的细粒度控制**
- 这是 Research Preview 阶段的限制，未来可能改进

> **对管理员的含义**：你只能决定整个组织"用不用 Cowork"，不能说"只给工程团队用"或"只给 Manager 以上用"。

---

### 2. 插件管理（企业级）

Team 和 Enterprise 计划的用户可以：

| 操作 | 说明 |
|------|------|
| 访问预装插件 | Sales、Legal、Data Analysis、Finance、Productivity 等 |
| 安装额外插件 | 从 Anthropic 公共仓库安装 |
| 自定义现有插件 | 在本地机器上修改 |
| 创建新插件 | 在本地机器上创建 |

**关键限制**：

| 能力 | 当前状态 |
|------|----------|
| 插件存储 | 本地保存在用户机器上 |
| 集中分发 | ❌ 不支持 |
| Admin 统一管理 | ❌ 不支持 |
| 组织级插件配置 | 🔜 规划中 |

**控制关系**：插件和 Cowork 使用同一个管理开关——没有单独的插件管理设置。关闭 Cowork = 同时关闭插件。

> **实际影响**：如果你的团队想统一使用某个定制插件，现在只能手动分发（比如通过 Slack 发文件），等未来 Anthropic 推出 Admin 集中管理功能。

---

### 3. 监控与合规限制

**这是企业用户必须注意的重要限制**：

| 合规能力 | 状态 | 影响 |
|----------|------|------|
| 对话历史存储 | 本地存储（用户电脑上） | 不受 Anthropic 数据保留期限约束 |
| Audit Logs | ❌ 不可用 | 无法审计 Cowork 活动 |
| Compliance API | ❌ 不可用 | 无法通过 API 提取合规数据 |
| Data Exports | ❌ 不可用 | 无法导出 Cowork 数据 |

> **严重警告（原文）**：**不要将 Cowork 用于受监管的工作负载**（Do not use Cowork for regulated workloads）。

**为什么这很重要**：
- 如果你的行业要求保留所有与 AI 交互的记录（如金融、医疗、法律），Cowork 目前不满足合规要求
- 对话记录在用户本地，如果用户删了或换电脑就没了
- 无法审计谁用 Cowork 做了什么——这对企业风控来说是一个盲区

---

### 4. 用量管理

**核心事实**：Cowork 任务消耗的用量**远超**普通 Chat。

**为什么消耗大**：
- 多步骤任务是计算密集型的，需要更多 Token
- 每次文件读写都需要上下文
- 并行 Sub-Agent 意味着**同时消耗多个模型实例的 Token**

**配额机制**：
- 用量配额**每 5 小时重置一次**（不是每天）
- 在 Settings > Usage 中监控个人用量

**用量管理技巧**：

| 策略 | 说明 |
|------|------|
| 合并相关任务 | 把相关工作放进同一个会话 |
| 简单问题用 Chat | 不需要文件访问的简单问题别用 Cowork |
| 监控用量 | 在 Settings > Usage 中查看 |
| 排队批处理 | 把相关任务攒在一起批量处理 |
| 具体描述 Prompt | 越具体 Claude 越少做无谓探索，越省 Token |

> **感性理解**：一个 Cowork 会话可能消耗相当于**几十条普通 Chat 消息**的用量。如果你的配额告急，先想想这个任务是不是非得用 Cowork。

---

### 5. 当前功能限制

Cowork 是 Feature Preview，以下功能**尚不可用**：

| 限制 | 详情 |
|------|------|
| **无 Projects 支持** | 不能在 Projects 中使用 Cowork |
| **无跨会话记忆** | Claude 不记得之前 Cowork 会话的内容 |
| **无会话共享** | 不能与他人分享 Chat 或 Artifact |
| **无 GSuite 支持** | 不兼容 GSuite 连接器 |
| **仅 macOS Desktop** | 不支持 Windows、Web、Mobile；不跨设备同步；Windows 版规划中 |
| **会话持久性** | 应用必须保持打开，关闭应用 = 结束会话 |

> **最关键的限制**：
> - **无跨会话记忆**：每次启动 Cowork 都是从零开始，Claude 不记得上次你让它做了什么
> - **仅 macOS**：Windows 用户暂时完全无法使用

---

### 6. 常见问题排除

**问题一：Claude 停止工作了**
- **检查**：Claude Desktop 应用在整个任务期间是否一直开着？
- **原因**：如果应用被关闭或电脑进入睡眠，会话可能已经结束了
- **解决**：重新启动任务

**问题二：用量配额很快就用完了**
- **原因**：Cowork 消耗远超普通 Chat
- **解决**：简单任务用 Chat，只有复杂的多步骤文件工作才用 Cowork

**问题三：文件没出现在预期位置**
- **检查**：是否授权了正确的文件访问权限
- **检查**：Claude 在完成任务时指定的输出位置是什么
- **解决**：确认权限和路径后重试

**问题四："Cowork is updating" 消息**
- **这是正常的**：表示 Cowork 正在更新到最新版本以应用修复和改进
- **不需要做什么**：等待更新完成即可

## 重点标记

1. **Admin 控制只有全局开关**：不能按用户/角色分配，要么全组织开要么全关
2. **不要用于受监管工作负载**：无 Audit Log、无 Compliance API、对话存本地——合规要求无法满足
3. **配额每 5 小时重置**：不是每天，规划任务时注意节奏
4. **一个 Cowork 会话 ≈ 几十条 Chat**：用量消耗差距巨大，简单问题用 Chat
5. **无跨会话记忆**：每次都从零开始，Claude 不记得上次会话
6. **仅 macOS**：Windows 版在规划中但还没出来
7. **应用必须保持打开**：这是最影响体验的限制——不能关掉电脑等结果
8. **插件无法集中管理**：企业目前只能手动分发自定义插件
