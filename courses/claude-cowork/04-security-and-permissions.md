# 模块四：安全模型与权限控制

> 对应 PDF 第 10-12 页

## 概念讲解

### 1. VM 架构安全机制

**技术基础**：Cowork 在你的电脑上运行一个虚拟机（VM），使用 Apple 的 **VZVirtualMachine（Virtualization Framework）**。

**沙箱机制**：
- 下载并启动一个**定制 Linux 根文件系统**
- 你授权的文件被挂载到 VM 内的沙箱路径：`/sessions/[session-name]/mnt/[folder]`
- Claude 的所有操作都在这个隔离容器内执行

**安全收益**：

| 特性 | 说明 |
|------|------|
| **受控环境** | Claude 在定义好的边界内操作，文件和网络访问都是受控的 |
| **隔离性** | VM 环境与你的主操作系统完全分离 |

> **但要注意**：VM 提供了隔离，但 Claude 确实能访问你授权的本地文件。审查 Claude 的计划后再放行，特别是处理敏感文件时。

---

### 2. 权限控制体系

Cowork 的权限机制与 Chat 模式保持一致。你能控制三个维度：

**（1）MCP 连接权限**
- 你决定连接哪些 MCP（Model Context Protocol）服务器
- 你决定它们多频繁地请求许可

**（2）网络访问权限**
- 通过设置控制 Claude 的网络访问
- Cowork 遵守你当前的网络出站权限设置（Network Egress Permissions）

**（3）文件访问权限**
- 你控制 Claude 可以访问哪些本地文件
- Claude 能读取、写入、**永久删除**这些文件（但删除有额外确认）

> **核心原则**：在扩展 Claude 默认设置之前，仔细评估你对某个 MCP 或网站的信任程度。信任是分级的——不是"全信"或"全不信"。

---

### 3. Prompt Injection 风险（首要安全威胁）

**定义**：Prompt Injection（提示注入）是 Cowork 面临的**首要安全问题**。恶意指令可以隐藏在 Claude 访问的网站、邮件或文档中，试图劫持 Claude 的计划，让它执行非预期操作。

**攻击原理**：
```
你给 Claude 一个正常任务
    → Claude 去访问某个网页/文档
        → 该网页/文档中嵌入了恶意指令
            → 恶意指令试图让 Claude 改变行为
                → 可能导致 Claude 执行非预期操作
```

**主要攻击向量**：
- **Web 内容**：是 Prompt Injection 的首要载体
- **邮件**：可能包含隐藏的恶意指令
- **文档**：看似正常的文档中可能嵌入攻击

> **特别警告**：如果你在使用 Claude in Chrome 扩展 + Cowork，限制只访问你信任的网站。Anthropic 强烈建议**不要**用 Claude in Chrome 来管理或操作涉及敏感信息的任务。

---

### 4. 文件访问安全实践

因为 Claude 能读取、写入和永久删除你授权的文件，必须谨慎授权。以下是推荐做法：

| 实践 | 说明 |
|------|------|
| **使用专用工作文件夹** | 不要把 Claude 指向整个 Documents 或 Desktop 文件夹 |
| **备份重要文件** | 授权 Claude 访问之前先备份 |
| **默认禁止删除** | 除非你明确需要删除，否则永远说 "don't delete anything" |
| **使用版本控制** | 用 git 管理的或项目专用的文件夹 |
| **审查计划** | 让 Claude 执行操作之前先审查它的方案 |

> **最佳实践**：创建一个专门用于 Cowork 的工作文件夹，只把需要处理的文件放进去。这样即使出了问题，影响范围也是可控的。

---

### 5. 网络与扩展安全

**网络安全**：
- Claude 的默认网络访问是**故意受限的**
- 只将访问权扩展到你信任的网站
- 不要随意放开网络限制

**扩展（MCP）安全**：
- Desktop 扩展（MCP）扩展了 Claude 的能力，但每个扩展都引入了新的攻击面
- 只使用 Claude Desktop 目录中经过验证的扩展
- 安装前仔细评估扩展请求的权限

> **思考框架**：每安装一个 MCP，就等于多开了一扇门。每扇门都可能成为攻击入口。只开必要的门，并且知道每扇门通往哪里。

---

### 6. Anthropic 的安全措施

Anthropic 在两个层面实施安全防护：

**（1）模型训练层**
- 使用**强化学习（Reinforcement Learning）**训练 Claude 识别和拒绝恶意指令
- 即使恶意指令看起来很权威或很紧急，Claude 也被训练来拒绝

**（2）内容分类器层**
- 所有进入 Claude 上下文的不可信内容都会被**扫描**
- 潜在的注入攻击会在影响行为之前被**标记**

**但要注意**：

> **Anthropic 原文警告**：虽然这些安全措施降低了风险，但攻击成功的概率仍然不是零（non-zero）。使用 Cowork 时始终保持谨慎。

这意味着：安全措施是"降低风险"而非"消除风险"。不能因为有安全措施就放松警惕。

---

### 7. 识别可疑行为

如果 Claude 出现以下行为，**立即停止任务**：

- 突然开始讨论无关话题
- 尝试访问意外的资源
- 未经提示就要求敏感信息

**发现可疑行为后**：
- 立即停止任务
- 报告给 `safety@anthropic.com`
- 或使用应用内反馈按钮

> **核心判断标准**：Claude 的行为是否偏离了你给它的任务？如果它突然做了你没要求的事，那就是红旗。

## 重点标记

1. **Prompt Injection 是第一大安全威胁**：恶意指令可能隐藏在 Claude 访问的任何外部内容中
2. **Web 内容是首要攻击向量**：Claude in Chrome + Cowork 的组合要格外小心
3. **VM 隔离 ≠ 绝对安全**：VM 隔离了执行环境，但授权文件仍可被真实修改
4. **专用文件夹是最佳实践**：不要把整个 Desktop/Documents 给 Claude，用专门的工作文件夹
5. **安全措施降低风险但不消除风险**：Anthropic 明确说攻击概率 non-zero
6. **每个 MCP 扩展 = 一个新的攻击面**：只装必要的、经过验证的扩展
7. **异常行为 = 立即停止**：Claude 做了你没要求的事就是红旗，不要犹豫直接停
