## 概念讲解

### 1. 系统要求（Requirements）

使用 Cowork 需要满足三个前提条件：

| 要求 | 详情 | 备注 |
|------|------|------|
| Claude Desktop 应用 | 必须是 macOS 版 | Windows/Web/Mobile 均不可用；Windows 版已规划 |
| 付费订阅 | Pro、Max、Team、Enterprise | 免费用户无法使用 |
| 网络连接 | 整个会话期间都需要保持连接 | 断网 = 任务中断 |

> **重要限制**：任务运行期间 Claude Desktop 应用必须保持打开状态。关闭应用 = 会话结束。这意味着你不能关掉程序去睡觉等结果。

---

### 2. 访问与操作流程

5 步启动一个 Cowork 任务：

```
1. 打开 Claude Desktop for macOS
2. 找到模式选择器 — 你会看到 "Chat" 和 Cowork 标签页
3. 点击 Cowork 标签页切换到 "Tasks" 模式
4. 描述你希望 Claude 完成的任务
5. 审查 Claude 的方案，然后让它运行
```

**注意区分两种模式**：
- **Chat 模式**：传统一问一答
- **Tasks 模式（Cowork）**：委托任务，Agent 自主执行

---

### 3. 任务执行体验

在任务运行过程中，你能获得以下体验：

**（1）进度指示器（Progress Indicators）**
显示 Claude 在每一步正在做什么。你不需要猜测——能实时看到进展。

**（2）透明度（Transparency）**
Claude 会展示它的推理和方案，让你能跟上它的思路。

**（3）实时介入（Steering）**
你可以随时跳进来纠正方向或提供额外指示。不是"启动后完全失控"，而是"你选择在哪个粒度介入"。

**（4）并行工作（Parallel Work）**
对于复杂任务，Claude 可能会协调多个 Sub-Agent 同时工作。

**（5）删除保护（Deletion Protection）**
Cowork 在永久删除任何文件之前都会要求你的**明确许可**。你会看到一个权限提示，必须选择 "Allow" 才能执行删除操作。

> **这是一个关键安全设计**：即使你授权了文件夹访问，Claude 也不会擅自删除文件。删除操作有额外的确认门槛。

**任务时长**：取决于复杂度，可以运行很长时间。你可以监控进度，也可以离开等 Claude 完成。

---

### 4. 插件系统概述（Plugins）

**定义**：Plugin（插件）是预打包的功能包，可以根据你的角色、团队和公司来定制 Claude 在 Cowork 中的工作方式。每个插件把以下元素组合在一起：

- **Skills**（技能）
- **Connectors**（连接器）
- **Slash Commands**（斜杠命令）
- **Sub-agents**（子代理）

简单说，插件就是一个"角色扮演包"——装上 Sales 插件，Claude 就知道怎么做销售相关的工作。

---

### 5. 官方插件库（11 个内置插件）

Anthropic 提供了一套覆盖常见知识工作的官方插件：

| 插件 | 用途 |
|------|------|
| **Productivity** | 任务管理、日历、日常工作流 |
| **Enterprise Search** | 跨公司工具和文档搜索信息 |
| **Sales** | 研究潜在客户、准备交易、跟进销售流程 |
| **Finance** | 分析财务数据、构建模型、跟踪关键指标 |
| **Data** | 查询、可视化、解读数据集 |
| **Legal** | 审查文档、标记风险、跟踪合规 |
| **Marketing** | 起草内容、规划活动、管理发布 |
| **Customer Support** | 分类工单、起草回复、提供解决方案 |
| **Product Management** | 编写规格、优先排列路线图、跟踪进度 |
| **Biology Research** | 文献搜索、分析结果、规划实验 |
| **Plugin Create** | 元插件——帮你从零创建自定义插件 |

**源码开放**：所有官方插件的完整源代码都在 GitHub 上公开：
`https://github.com/anthropics/knowledge-work-plugins`

> **亮点**：Plugin Create 是一个"创建插件的插件"——如果现有插件不满足你的需求，你可以让 Claude 帮你做一个。

### 6. 插件安装与自定义

#### 安装插件

两种方式：

**方式一：图形界面**
```
1. 打开 Claude Desktop for macOS
2. 进入 Cowork 标签页
3. 点击左侧边栏的 "Plugins"
4. 浏览可用插件并点击安装
5. 也可以点击 "Upload plugin" 上传自定义插件文件
```

**方式二：终端命令**
```bash
claude plugins add knowledge-work-plugins/sales
```

#### 自定义插件

```
1. 查看已安装的插件时，点击右上角的 "Customize" 按钮
2. 系统会自动生成一个 Cowork prompt 来定制该插件
3. 点击 "Let's go" 开始与 Claude 一起定制
```

> **注意**：插件目前保存在本地机器上。组织级别的插件统一分发功能还在规划中（未来更新）。

---

### 7. 插件使用方式

安装插件后，每个插件都会添加可调用的命令。使用方式：

- 输入 `/` 查看可用命令
- 或者点击 `+` 按钮浏览已安装插件的命令列表

**适用场景**：
- 需要特定领域工作流程时（如销售、法务、数据分析）
- 想让 Claude 按照特定角色的规范来工作
- 团队内统一工作流标准

## 重点标记

1. **仅 macOS Desktop 可用**：Web、Windows、Mobile 都不行，Windows 已规划
2. **应用必须保持打开**：关掉 Claude Desktop = 会话终止，这是当前最大的体验限制
3. **删除保护是硬性安全措施**：任何文件删除都需要你明确授权
4. **插件 = 角色包**：把 Skills + Connectors + Commands + Sub-agents 打包成一个可安装的功能集
5. **Plugin Create 是元插件**：如果现有 11 个插件不够用，Claude 可以帮你从零创建
6. **插件目前仅限本地**：不支持组织级别统一分发，Admin 无法集中管理（规划中）
7. **源码开放**：所有官方插件代码都在 GitHub 上，可以直接查看和修改
